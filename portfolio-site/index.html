AI Test Studio

AI Test Studio is a new unified testing UI driving a fundamental shift in software testing from fragmented, skill‑dependent processes to an AI‑driven, autonomous testing fabric that unifies functional, performance, UX, accessibility, and API testing into a single workflow. It embodies the next evolution of Perforce’s AI‑driven “shift‑left” testing vision.

Research

The problem

Research

There are different products for quality assurance which Perforce offers. The problem is that these products are complex and it can be difficult to use these testing applications for junior testers or even for people who don’t know how to write tests. Also there can be multiple variants of one test in these applications. This cause losing of potential customers.

Workshops, interviews with participants and observations

Research

We had several meetings with stakeholders where we discussed the problem, ideas and our visions. It was also important to start with initial UX research. We asked participants from DPP program initial questions like what type of testing they do, what are theirs problems, what are theirs workflows, what are paint points of product which they use etc. We also sent several surveys to these participants to get quantitative data. We wanted to know which environments participants use for testing, how many participants automate their testing etc. We also introduced all participants our vision and we wanted to know what they think about this idea and if they want to start using product which we were thinking about.

Business goals

Research

Main business goal was to remove technical barrier of writing testing scripts and involve users in testing of theirs applications. AI test studio connects together existing Perforce applications like BlazeMeter and Perfecto. It’s expected that customers of BlazeMeter will start using Perfecto and vice versa.

User needs

Research

After interviews with potential users of the application I defined that main user needs. User needs are ability to create tests without knowledge how to write tests, maintain tests in one place and possibly to run functional and performance test from one test definition.

Personas

Research

I defined after interviews with potential users following personas:

Persona 1: The Quality Engineering Leader(Head of QA, Director of QE, Test Manager)

Who they are

  • Own quality strategy and tooling across web, mobile, APIs, and performance
  • Often existing Perfecto and/or BlazeMeter customers, or using similar tools
  • Manage teams of automation engineers + manual testers; report into VP Eng / CTO

Key Goals

  • Standardize tooling and reduce test fragmentation (UI, API, mobile, performance)
  • Shorten regression cycles and stabilize releases across teams
  • Increase test automation coverage without ballooning headcount
  • Demonstrate quality impact to execs with clear metrics and reports

Core Pain Points

  • Multiple, siloed tools (separate for mobile, web, API, performance)
  • High maintenance of test suites; flaky tests and brittle scripts
  • Difficulty getting non-technical testers/product folks to contribute to automation
  • Lots of time spent collating reports from different systems into one view

Why Simo / AI Test Studio resonates

  • Unified dashboard for all testing activities → aggregate mobile (Perfecto), web, API, performance (BlazeMeter)
  • No‑code test creation wizards → enable manual testers and BAs to contribute
  • Integrated reporting system + comprehensive analytics and insights → single, shareable view for leadership
  • Real-time test execution monitoring → quicker detection of issues during runs

Buying Triggers

  • Tool consolidation or standardization initiative
  • Pressure to increase coverage and speed without adding headcount
  • Recent production issues tied to gaps or fragmentation in testing

Ideation

Research

First idea how to help testers was to involve AI chat to theirs testing flow. This helps junior testers start using BlazeMeter and Perfecto. Also we wanted to offer single source of true and solve another problem which was having tests in different places. We wanted to created simple modern AI oriented testing application which will be oriented for junior testers or on people who can’t write testing scripts.

User flows

Research

In one of the first meeting we defined together first user flow for new user and sketched on the board which screens we might need. I continued with defining more detailed user flows after that until we all agreed on the application flow. I defined two main flows. One for new user and one for existing user.

Research

There are many different entities in this projects like: suites, test scenarios, environments, policies. It was important to define solid information architecture. Research method was interviews with internal stakeholders.

UI design

Validate solution

User testing

Research

The goal — to conduct usability tests for the MVP of AI Test Studio with the DPP participants as well as internal participants. We wanted to understand the UX workflows and paint points and how the participants interact with the chat flow.

Test script — we didn’t create any particular user testing script, but we wanted to understand how the users understand the product. So we let them to explore the application and we observed their steps in the application.

Findings — we created document based on testing of each participants and than from these data we identified issues and consolidated them. Consolidated list of the issues that the users came across and were observed in the usability test:

  • Onboarding & Initial Guidance Issues
  • Terminology & Mental Model Mismatches
  • AI Chat Interaction and Workflow Problems
  • Chat UI / Integration Friction
  • Test Creation and Management
  • Test Execution and Configuration difficulties
  • Results visibility and Analysis limitation
  • History Management and Privacy

Usability issues workshop

Research

We conducted usability issues workshop. The goal was to prioritise usability issues.

My learnings

Changing direction

We found out during user testing, that some users were lost in classic flow. There are different terms like suites, policies, environments. These brings cognitive load for some users and AI chat can help with this. We decided to focus on AI part of the application which should help users with this obstacle.

AI

I learned how to design AI chat. It was my first product which was AI based. To design AI product requires completely new skills set in my opinion.
